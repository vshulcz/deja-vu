How it works

deja parses the local history stores each harness already writes (~/.claude/projects, ~/.codex/sessions, opencode's SQLite db) into an incremental inverted index in ~/.cache/deja. Only changed files are re-read on each run. The MCP server exposes the same index to agents. Also: secrets are redacted at ingest, deja stats summarizes your agent work, deja share hands a sanitized session to a colleague, and deja sync moves memory between machines. Details in ARCHITECTURE.md — adding a harness is one parser file.
