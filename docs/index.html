your agents already solved this. deja finds it.

Memory tools start empty and record forward. deja starts full: it indexes the sessions your coding agents already wrote to disk, searches gigabytes in ~12 ms, serves memory back over MCP, and moves with you between machines over SSH. One zero-dependency binary, fully local.

$ deja --help

deja "query"
search everything your agents ever wrote — phrases, fuzzy, semantic tiers. Retroactive: months of logs from before you installed it.
recall / recall_context
MCP tools, answers under ~4 KB. A fix found in Codex is recalled inside Claude Code.
deja blame <path>
which sessions touched this file, what was decided and why. git blame, but for the discussions behind the code.
deja handoff --to codex
stuck in one agent? package the live context and continue in another.
deja remember "text"
keep durable decisions next to the recalled history — same index, same redaction.
deja sync ssh laptop
memory follows you between machines. Watermarked, idempotent, no cloud in the middle.
deja share <id>
hand a colleague a sanitized digest — problem, key conclusions, secrets already scrubbed.
deja stats
sessions, top projects, activity sparkline. Your agent year, one screenshot.

Secrets never reach the index: API keys, JWTs and private-key blocks are stripped at ingest — the cache is safe to keep, shares and exports are safe to send.

$ man deja # how it works

deja parses the local history stores each harness already writes (~/.claude/projects, ~/.codex/sessions, opencode's SQLite db) into an incremental inverted index in ~/.cache/deja. Only changed files are re-read on each run. The MCP server exposes the same index to agents. Secrets are redacted at ingest. Details in ARCHITECTURE.md — adding a harness is one parser file.
